Hugo + Spacemacs

十月 1, 2019

网上相关资料较少，实在忍受不了国内的博客互相复制粘贴(人类的本质)但是有用的信息很少的现状

我将自己的踩坑历程分享在这里，如果有问题，欢迎大家批评指正

仍在学习阶段,水平不足,请大家见谅

概述

系统环境

Manjaro KDE

[windows 环境下还没有尝试，windows 安装各种东西很复杂，需要设置各种环境变量和文件路径，已弃坑]

Windows 环境下，建议使用 git 来进行操作(已集成 ssh) 或通过 msys2 虚拟环境，或 WSL 中安装 linux，修改 cmder 启动项替代 windows 的 powershell(后续可能会提供相应的踩坑经历)

Hugo

简述

Hugo 是什么

Hugo 是由 Go 语言实现的静态网站生成器。简单、易用、高效、易扩展、快速部署。

同类型的有 Hexo,Jekyll,WordPress 等,其中 hugo 的生成速度比较快,依赖较少,故而推荐使用 Hugo 如果想要拥有更炫酷的主题,可以考虑使用 Hexo,速度慢于 Hugo

使用 spacemacs 来编写博客

  • 可以使用任何的支持 markdown 文件格式的编辑器来编写博客,如 vs code ,sublime,Typora 等

  • 对 spacemacs 有特殊的感情(毕竟已经使用了很久了),spacemacs 配合 evil-mode 以及丰富的快捷键(根本记不住)可以实现快捷的操作

emacs 的 org-mode 很强大,配合 latex 可以完成论文或小型笔记的需求,可以插入图片(不是很方便), emacs 几乎可以实现你的任何需求

  • 总之,挑一个 顺手 的编辑器

Hugo 部署方案

可以选择将自己的博客部署在自己的服务器上,但是穷学生经济实力有限,所以选择将 hugo 部署在 github 上

Hugo 安装及配置

hugo 安装(使用包管理器 yay)

需要在 manjaro 中安装 yay,命令为 sudo pacman -S yay yay -S hugo (注意,使用 yay 命令时不要加 sudo)

本地博客仓库创建

  • 创建本地博客仓库,命令为 hugo new site example ,example 处填上自定义名称

安装主题

Hugo themes 官网为 https://themes.gohugo.io/

  • 需要将对应的 theme 从 github 上 clone 到本地博客仓库的 themes 文件夹中

  • 在这里我使用的 themes 是 Tranquilpeak,过程如下 

  cd localblog/themes
  git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git

  #localblog处填本地仓库的地址

  • 在本地仓库的配置文件中修改 theme = "主题所在文件夹的名称"即可

  • 例,在这里 theme = "hugo-tranquilpeak-theme"

修改配置文件

各个主题都有推荐的配置文件，即模板配置，一般保存在相应的 yourTheme/exampleSite 中,可在该配置的基础上进行修改 不同的主题可能会有不同的博客地址，如 content/post　或　content/posts,需要根据不同的主题修改对应的 hugo 配置，以及 easy-hugo 配置 使用推荐配置的过程如下

  • 用 yourTheme/exampleSite 文件夹下的配置文件 config.toml 复制到博客仓库下,用于替换原有的配置文件

  • 根据 theme 的官网指导修改 config.toml,注意 baseURL 处填写博客域名,例 baseURL = "https://WanyouWang.github.io"

  • 根据需要修改博客标题,修改作者信息,开放评论区和链接共享区,在这里需要注意保存图像的文件夹,在博客的 static 区和 yourTheme/images 区都有保存图像的文件夹,其中 static 区文件夹优先级高于 yourTheme/images

github 部署及配置

在 github 上创建个人项目

项目名称应为 githubname.github.io(githubname 中填写你的的 github 名称,不可填写其他名称) 通过访问 githubname.github.io，观察项目是否创建成功

git ssh 连接

可以使用 ssh 秘钥直接与 github 进行链接，避免重复输入密码，方便后续利用脚本一键部署

安装 ssh

arch linux 下 ssh 开源管理软件为 openssh 安装命令如下 yay -S openssh

生成 ssh 秘钥

生成 ssh 秘钥的操作为 

  ssh-keygen -t rsa -C "your_email@example.com"
  #"your_email@example.com"处填写你的邮箱地址
  #接下来按照要求输入公钥的名字,例如id_rsa,接下来输入私钥的密码(需要记住)
  #将ssh私钥添加到ssh-agent
  ssh-add ~/.ssh/id_rsa

注意 xxx.pub 是公钥，可以给出 xxx 是私钥，不可泄露

将 ssh 私钥与 github 相关联

  • 登录 github,进入 settings -> SSH and GPG keys

  • 在 SSH keys 处,选择 new SSH keys

  • 上传上一步生成的私钥并保存

  • 在本地测试连接 ssh -T git@github.com,如果出现下述文字则说明 ssh 配置成功 

    " Hi xxx! You've successfully authenticated, but GitHub does not provide shell access."

编辑 ~/.ssh/config 文件(中英文对齐）

当有多个账户需要管理时，可在 config 文件中设置 host 别名和主机地址(若该文件不存在,则创建一个新的文件) 此文件在 spacemacs easy-hugo github 一键部署中也需要使用 , 填写内容如下 例 

Host blog.github.com  
    HostName github.com  
    User git  
    IdentityFile ~/.ssh/id_rsa
Host 主机别名，用户登录时需要
HostName 主机名，远程连接的服务器
User
IdentityFile 用户登录私钥

这时可以通过访问 blog.github.com 来访问你的 github,可以通过 ssh -T blog.github.com 进行访问测试

使用 hugo 编写发布博客

编写博客

  • 在本地博客仓库下,创建新博文,命令为 hugo new post\xxx.md 或 hugo new posts\xxx.md (根据不同主题选择是 post 文件夹还是 posts 文件夹)

  • 这里使用 ox-hugo 将 org 文件转换为 md 文件,再使用 md 文件进行发布(也可以直接编写 markdown 文件)

  • hugo 原生支持 org,可以直接发布 org 文件,但是支持不是很好,有时候排版会出现一些小 bug

注意博客的编写完成情况 script: true 草稿状态,可以预览,不会发布 script: false 完成状态，可以预览和发布

预览博客

在博客的根目录下,需要在本地启动 hugo 服务器，命令为 hugo server -D 服务器启动后,浏览器中输入 localhost:1313 进行预览

发布博客

  • 在博客的根目录下，编译博客，命令为 hugo (如需使用配置文件外的主题，可在 hugo 后添加主题信息),编译完成后，编译结果保存在 public 文件夹中

  • 我们的博客部署在 github 上,将 public 文件夹中的内容上传 github 的博客仓库中,博客会自动链接，此后可访问自己的博客域名观察结果

  • 这里可以通过常规的操作使用 git,如 github desktop,git 命令等,也可以使用 emacs 的 magit 插件,以及后续的 一键部署脚本

spacemacs 配置

所需 layer

org ox-hugo easy-hugo

ox-hugo 配置

ox-hugo 不能直接通过 spacemacs 的 addition-packages 安装，需要根据官网给出的条件进行配置，或者在个人配置中添加下列语句 (setq ox-hugo-enable-support t) 系统会自动安装 ox-hugo

在当前的博文开头段,可以使用以下模板 

# -*- mode: snippet -*-
# name: hugo
# key: h
# --
#+HUGO_BASE_DIR: ../../
#+TITLE: $1
#+OPTIONS: toc:2 \n:t
#+DATE:`(format-time-string "<%Y-%m-%d %H:%M>" (current-time))`
#+HUGO_AUTO_SET_LASTMOD: t
#+HUGO_TAGS: $2
#+HUGO_CATEGORIES: $3
#+HUGO_DRAFT: false

$0

  • 其中 HUGO_BASE_DIR 用于 org -> md 文件时,确认文件生成的路径

  • OPTIONS 可以自定义 toc:2 指 Table of Contents 支持到二级标题

  • 可以在 org-mode 中开启 org-hugo-auto-export-mode,可以自动将 org 转换为 md 文件

  • 在 org-mode 下,快捷键 , e e H h 可以手动将 org 文件导出为 md 文件

easy-hugo 配置

easy-hugo 各变量及其含义

easy-hugo-basedir hugo 本地仓库地址
easy-hugo-postdir hugo 主题对应的博客存放地址(根据主题不同,为 post 或 posts 文件夹
easy-hugo-url 博客地址
easy-hugo-sshdomain 使用的 ssh 别名
easy-hugo-root 编译后的博客的存放地址
easy-hugo-default-ext easy-hugo 默认博客文件类型(默认为.md)

例 

  (setq easy-hugo-basedir "~/Documents/Blog/Melancholy/"
        easy-hugo-postdir "content/post"
        easy-hugo-url "https://wanyouwang.github.io/"
        easy-hugo-sshdomain "blog.github.com"
        easy-hugo-root "~/Documents/Blog/Melancholy/public/"
        easy-hugo-default-ext ".org"
        )

evil-mode 下重新定义键位

spacemacs layer 结构

Readme.org
packages.el 列出需要的包以及配置包(可使用 use-package)
funcs.el 自定义函数
config.el 自定义配置

在 funcs.el 添加自定义函数(函数名可自定义，也可使用 layer_name/function_name)，用于修改 easy-hugo 在 evil-mode 中的键位定义 注意　evil-define-key 和 list xxx mode 

  (defun easy-hugo-keymaps ()
    (interactive)
    (evil-define-key
      (list 'normal 'insert 'visual 'motion)
      easy-hugo-mode-map
      "n" 'easy-hugo-newpost
      "D" 'easy-hugo-article
      "p" 'easy-hugo-preview
      "P" 'easy-hugo-publish
      "o" 'easy-hugo-open
      "d" 'easy-hugo-delete
      "e" 'easy-hugo-open
      "c" 'easy-hugo-open-config
      "f" 'easy-hugo-open
      "N" 'easy-hugo-no-help
      "v" 'easy-hugo-view
      "r" 'easy-hugo-refresh
      "g" 'easy-hugo-refresh
      "s" 'easy-hugo-sort-time
      "S" 'easy-hugo-sort-char
      "G" 'easy-hugo-github-deploy
      "A" 'easy-hugo-amazon-s3-deploy
      "C" 'easy-hugo-google-cloud-storage-deploy
      "q" 'evil-delete-buffer
      (kbd "TAB") 'easy-hugo-open
      (kbd "RET") 'easy-hugo-preview)
    )

github 一键部署

easy-hugo 建议使用脚本一键部署 在 Hugo 根目录下,创建脚本 注意脚本创建完成后，需要修改脚本的运行权限，否则不能直接执行 chmod +x ~/xxxblog_folder/deploy.sh

配置 1 

  #!/bin/bash
  echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

  # Build the project.
  hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

  # Go To Public folder
  cd public

  git init
  git remote add origin git@github.com:WanyouWang/WanyouWang.github.io.git
  # Add changes to git.
  git add .

  # Commit changes.
  msg="rebuilding site `date`"
  if [ $# -eq 1 ]
    then msg="$1"
  fi
  git commit -m "$msg"

  # Push source and build repos.
  git push origin master -f

  # Come Back up to the Project Root
  cd ..

  rm -rf public

该脚本会从博客远程仓库下载之前的博客项目，并对其进行修改 不容易出现问题，但是不会记录之前的博客版本 注意 git remote 处需要添加自己的博客 ssh 地址

官网配置 

  #!/bin/sh

  # If a command fails then the deploy stops
  set -e

  printf "\033[0;32mDeploying updates to GitHub...\033[0m\n"

  # Build the project.
  hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

  # Go To Public folder
  cd public

  # Add changes to git.
  git add .

  # Commit changes.
  msg="rebuilding site $(date)"
  if [ -n "$*" ]; then
      msg="$*"
  fi
  git commit -m "$msg"

  # Push source and build repos.
  git push origin master
